# Module 8: Secure systems are easier to operate safely

SRE interviews often include security-adjacent questions because reliability and security overlap heavily in real systems:

- secrets leakage can become an availability incident
- poor access controls create massive blast radius
- weak rollout controls can spread vulnerable builds quickly

## Shift-left is not enough

Security work is not finished at code review. Operationally strong teams also rely on:

- secret scanning
- short-lived credentials where possible
- managed secret delivery
- access scopes that match the task
- auditability and rotation playbooks

## What interviewers want to hear

When a credential leaks, a strong answer usually includes:

1. treat it as compromised
2. contain by rotating or revoking it
3. determine exposure and dependent systems
4. communicate clearly
5. add controls so the same class of mistake is less likely next time
