# Module 9: Efficiency is part of operating a healthy system

FinOps questions show up because mature SRE work is not only about "keeping it up." It is also about keeping systems efficient enough to scale responsibly.

## Why SREs should care about cost

- overprovisioned systems hide design and capacity issues
- unnecessary spend limits future reliability investment
- efficiency changes are still operational changes and need guardrails

The right answer is not "cut cost at all costs." It is:

> "Use real usage data, keep SLOs in view, and make efficiency changes with rollback criteria."

## Common cost levers

- rightsizing compute and requests/limits
- storage lifecycle management
- turning off idle resources
- choosing the right tier for the workload
- better attribution so teams see what they own

## What makes a good explanation of cost-aware reliability

Strong candidates talk about:

- baseline measurement
- expected savings
- headroom for peaks
- service-level guardrails
- rollback triggers if performance regresses
