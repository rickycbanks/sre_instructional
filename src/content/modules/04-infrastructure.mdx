# Module 4: Infrastructure choices that make operations safer

Cloud names change; operational concepts do not. Interviewers are usually not checking whether you can recite every provider product. They care more that you can reason about:

- compute
- storage
- networking
- identity
- rollout safety
- repeatable infrastructure changes

## Think in abstractions first

If someone asks about AWS, Azure, or GCP, start by anchoring on the underlying need:

- "We need object storage for static assets."
- "We need a managed relational database."
- "We need container orchestration with autoscaling."

Then map that need to the provider-specific service. That is a much stronger signal than memorizing brand names.

## Why infrastructure as code matters

IaC is important because it gives you:

- repeatable changes
- reviewable diffs
- fewer console-only snowflakes
- faster recovery in new environments
- a shared source of truth

Strong answers also mention that infrastructure changes should be deployed with the same discipline as application code: review, testing, staged rollout when possible, and rollback plans.

## What interviewers want in launch architecture answers

For a major launch or traffic spike, a strong answer usually includes:

- edge protection such as caching or CDN
- dependency protection (timeouts, retries, circuit breakers)
- progressive rollout for risky changes
- observability and rollback criteria
- some acknowledgment of cost versus resilience tradeoffs
